# Milestone 2 Integration Plan

## Goal
Complete secure feature integration for authentication, authorization, file storage, validation, and error handling.

## Integration Order
1. Security baseline and middleware foundations
2. Google OAuth + JWT issuance and validation
3. Firebase Storage for file upload and secure retrieval
4. RBAC enforcement for role-restricted actions
5. Centralized error handling and standardized API responses
6. Feature-specific testing and documentation

## Implemented Components

### 1. File Storage and Retrieval (Firebase Storage)
- Backend profile upload route validates MIME type and max file size (2 MB).
- Files are stored in Firebase Storage under user-scoped object paths.
- Stored files are retrieved via short-lived signed URLs generated by the backend.
- Previous profile image objects are deleted during replacement to avoid orphan files.

### 2. User Authentication (Google OAuth + JWT)
- Google OAuth login is handled through Passport Google strategy.
- JWT is issued after local login/register and OAuth callback.
- JWT payload includes `userId` and `role`.
- Auth middleware validates token signature and expected payload fields.

### 3. RBAC and Middleware Validation
- Added `requireRole` middleware for role-gated routes.
- Admin-only routes:
  - `GET /api/auth/users`
  - `PATCH /api/auth/users/:id/role`
- Existing user routes remain protected by auth middleware.
- Request payloads and route params are validated via `express-validator`.

### 4. Centralized Error Handling
- Added unified not-found middleware.
- Added centralized error middleware with normalized responses:
  - Validation errors
  - Duplicate-key conflicts
  - Cast/ID errors
  - Generic server errors

### 5. Security Enhancements
- Helmet configured globally for secure headers.
- Session cookie configured with `httpOnly`, `sameSite`, and production `secure` flag.
- Required sensitive environment variables are enforced at server startup.
- OAuth redirect now places token in URL hash to reduce token leakage via query logs.

## Environment Variables Required
- `MONGO_URI`
- `JWT_SECRET`
- `SESSION_SECRET`
- `FRONTEND_URL` (recommended)
- `CORS_ORIGIN` (optional override)
- `GOOGLE_CLIENT_ID`
- `GOOGLE_CLIENT_SECRET`
- `FIREBASE_STORAGE_BUCKET`
- `FIREBASE_PROJECT_ID` (if using service account env setup)
- `FIREBASE_CLIENT_EMAIL` (if using service account env setup)
- `FIREBASE_PRIVATE_KEY` (if using service account env setup, with escaped newlines)

## Notes
- Firebase storage access is private and served through backend-generated signed URLs.
- RBAC is intentionally scoped to admin account-management endpoints for clear permission separation.
